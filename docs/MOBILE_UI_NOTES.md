# HomeStack UI Mobile Notes

Project Solace and Project Meridian should be designed mobile-first for common household use.

## Principles

- Primary actions must be reachable with one thumb.
- Use cards for common mobile lists; keep tables for desktop/power-user views.
- Minimum practical touch target: 44px high.
- Keep destructive actions visible but secondary.
- Put the most-used routes in a mobile bottom navigation bar.
- Avoid wide dense tables on phones unless there is also a card/list alternative.
- Use short page descriptions and clear empty states.

## Standard mobile navigation

Use a bottom tab bar for the most-used actions:

- Home
- Bills or Tasks
- Payday or Approvals
- Buckets or Rewards
- Calendar or Dashboard

For Meridian, the equivalent might be:

- Home
- Tasks
- Rewards
- Approvals
- Users

## Standard responsive behaviour

- Desktop: navbar + tables + multi-column dashboards.
- Tablet: navbar + cards/tables depending on page.
- Phone: bottom tab bar + card lists + full-width buttons.

## Visual direction

Keep the HomeStack warm minimal style: beige/cream light mode, warm charcoal dark mode, soft cards, muted accents, and clear spacing.
